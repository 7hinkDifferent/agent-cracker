# 05. 最小实现难度评估与落地步骤

## 目标定义（建议）

最小可用（MVP）建议先做到：
- Tool 层：`present/hide/navigate/eval/snapshot/a2ui_push/a2ui_reset`
- 渲染层：先复用 OpenClaw Canvas（减少前端容器成本）
- 交互层：至少支持 `openclawSendUserAction(...)` 回传到 agent

## 难度评估

- 仅做 tool 协议 + 复用 OpenClaw 渲染：**中等偏低**
- 自建渲染容器（含 bridge、安全、A2UI host）：**中等偏高**

主要复杂度来源：
- 节点选择和命令路由（node.invoke 兼容性）
- 多端行为差异（特别是 hide/present 语义）
- A2UI v0.8 与消息验证
- Action 回传的来源校验与安全边界

## 分阶段实施（推荐）

### Phase 1：协议层打通（1-2 天）

交付：
- 新增 `canvas` tool（动作/参数与 OpenClaw 对齐）
- 统一返回规范（`eval.result`、`snapshot` 图片）

验收：
- 能稳定调用 `canvas.present/navigate/eval/snapshot`

### Phase 2：A2UI 基础（1 天）

交付：
- `a2ui_push` 接受 `jsonl/jsonlPath`，支持快速文本构造
- JSONL v0.8 校验，识别 v0.9 并报错

验收：
- 推送 `surfaceUpdate + beginRendering` 可见
- `a2ui_reset` 可恢复初始状态

### Phase 3：Action 回传（1-2 天）

交付：
- 注入 `openclawSendUserAction` helper
- native/bridge -> agent 的事件转换（可先沿用 `CANVAS_A2UI ...` 文本协议）

验收：
- 页面按钮点击能触发新一轮 agent run

### Phase 4：安全与稳定性（1 天）

交付：
- 路由鉴权（token/capability）
- 路径白名单
- 超时、重试、错误码标准化

验收：
- 非授权请求不能访问 canvas host
- 恶意路径与非法 JSONL 被拒绝

## 粗略工时（单人）

- 最小 demo：3-5 天
- 可上线版本：5-8 天

## 风险清单

- 平台差异导致“看似成功但行为不同”（尤其 Android hide）。
- A2UI 版本混用（v0.8/v0.9）造成渲染失败。
- 如果不做来源校验，Action bridge 存注入风险。
