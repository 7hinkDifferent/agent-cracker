# 04. 安全模型与权限边界

## 1) Canvas host 访问控制（HTTP/WS）

关键文件：`projects/openclaw/src/gateway/server-http.ts:122`

Canvas 路由（`/__openclaw__/canvas`、`/__openclaw__/a2ui`、`/__openclaw__/ws`）会走专门鉴权：
- 本地直连（loopback/trusted proxy）可放行。
- 或 bearer token 认证通过。
- 或带有效 capability token（`oc_cap`）并且存在匹配的在线 node ws client。

## 2) capability token 机制

关键文件：
- `projects/openclaw/src/gateway/canvas-capability.ts:3`
- `projects/openclaw/src/gateway/server/ws-connection/message-handler.ts:781`

行为：
- node 连接时 mint token，并把 `canvasHostUrl` 改写成 scoped URL（含 `/__openclaw__/cap/<token>`）。
- 过期时间默认 10 分钟，命中时会滑动续期。
- URL 中 path scope 会被规范化重写到 query `oc_cap` 参与鉴权。

## 3) 节点端本地文件边界

### macOS custom scheme

关键文件：`projects/openclaw/apps/macos/Sources/OpenClaw/CanvasSchemeHandler.swift:8`

- session name 禁止 `/` 和 `..`。
- 请求文件解析后必须仍在 session root 内（目录穿越防护）。
- root 无 `index.html` 时回退 scaffold/welcome，不会任意读系统文件。

### agent tool 读取 jsonlPath

关键文件：`projects/openclaw/src/agents/tools/canvas-tool.ts:29`

- `a2ui_push` 的 `jsonlPath` 会经过 inbound path policy；超出允许 roots 直接拒绝。

## 4) 命令执行面收敛

关键文件：`projects/openclaw/src/gateway/node-command-policy.ts:5`

- `canvas.*` 在 node allowlist 中显式列举。
- gateway 会把 node 声明命令与 allowlist 交集后再暴露。
- 这能防止 node 私自暴露非预期命令被 agent 调用。

## 5) Action 来源校验

- iOS：`projects/openclaw/apps/ios/Sources/Screen/ScreenWebView.swift:183`
- macOS：`projects/openclaw/apps/macos/Sources/OpenClaw/CanvasA2UIActionMessageHandler.swift:22`

都做了来源限制：
- 仅接受本地 canvas scheme 页面，或局域网/本地网络可信 host 页面。
- 拒绝任意外部网页直接触发 native action handler。
