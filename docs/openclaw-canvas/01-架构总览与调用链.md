## 2) 核心链路（简化）

```text
Agent
  -> canvas tool (action)
  -> gateway: node.invoke(command, params)
  -> Node Runtime (macOS / iOS / Android)
  -> Canvas Controller / Manager
  -> WebView load/eval/snapshot
  -> payload 回传 gateway
  -> tool 返回给 Agent
```

## 3) 命令到渲染的落点

- `canvas.present`
  - macOS：走 `CanvasManager.showDetailed(...)`，支持 panel 位置尺寸。
  - iOS：忽略 placement，直接 `screen.navigate(...)` 或默认 canvas。
  - Android：当前实现实际是 `canvas.navigate(url)`。
- `canvas.hide`
  - macOS：隐藏 panel。
  - iOS：切回默认 canvas（不是“销毁窗口”）。
  - Android：当前 dispatcher 基本是 no-op（返回成功）。
- `canvas.navigate`：跳转 URL / 本地路径。
- `canvas.eval`：在 WebView 中执行 JS 并返回字符串化结果。
- `canvas.snapshot`：WebView 截图后返回 `{ format, base64 }`。

## 4) A2UI 链路（重要）

- `canvas.a2ui.pushJSONL` / `canvas.a2ui.push`
  - 节点端会先确保加载 A2UI 页面（`/__openclaw__/a2ui/`），再 `host.applyMessages(messages)`。
  - 当前强约束 v0.8 消息（不支持 v0.9 `createSurface`）。
- `canvas.a2ui.reset`
  - 调 `host.reset()`，常用于清状态。

## 5) Action 回传链路（Canvas -> Agent）

- 页面侧通过注入 helper 发送：`openclawSendUserAction(...)`。
- 原生侧 handler 名：`openclawCanvasA2UIAction`。
- 原生将 action 组装成结构化文本事件：`CANVAS_A2UI ... default=update_canvas` 并触发新的 agent run。

- iOS 入口：`projects/openclaw/apps/ios/Sources/Screen/ScreenWebView.swift:172`
- macOS 入口：`projects/openclaw/apps/macos/Sources/OpenClaw/CanvasA2UIActionMessageHandler.swift:8`
- 文本格式化：`projects/openclaw/apps/shared/OpenClawKit/Sources/OpenClawKit/CanvasA2UIAction.swift:66`
